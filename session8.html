01. NETWORK ATTACKS

Attacking protocols and traffic flow.

On-Path (Man-in-the-Middle)

Intercepting Traffic

The attacker places themselves between two communicating parties to eavesdrop or alter data.

  • ARP Poisoning: Redirecting Layer 2 traffic via fake MAC addresses.
  • IP Spoofing: Impersonating a trusted IP address.
  • Replay Attack: Capturing a valid session packet and re-sending it later.

"The victim thinks the connection is direct and secure."

DDoS & DoS Attacks

Botnets

Using "zombie" computers to flood a target with traffic (DDoS).

Exhausting Resources

Aiming to break Availability by overwhelming the system.

  • Amplification: Using small requests to trigger massive responses (e.g., DNS Amplification).
  • SYN Flood: Abusing the TCP 3-way handshake to hang connections.

DNS Poisoning & Hijacking

Manipulating Resolution

Corrupting the system that translates names (google.com) to IPs (8.8.8.8).

  • DNS Poisoning: Inserting fake entries into a DNS cache.
  • Domain Hijacking: Changing registration details to steal a domain.
  • URL Hijacking: Typosquatting (gogle.com).

"Targeting the phonebook of the internet."

02. APPLICATION ATTACKS

Exploiting vulnerabilities in software and code.

SQL Injection (SQLi)

' OR 1=1 --

The classic bypass.

Database Manipulation

Injecting malicious SQL commands into an input field to bypass authentication or dump the database.

  • Defense: Input validation and prepared statements.

Cross-Site Scripting (XSS)

Malicious Scripts

Injecting client-side scripts (JavaScript) into web pages viewed by other users.

  • Goal: Steal cookies, session tokens, or deface websites.
  • Defense: Output encoding and sanitization.

"Stored, Reflected, or DOM-based."

CSRF & Replay

CSRF

Forces a logged-in user to execute unwanted actions (e.g., changing password) on a web app they are authenticated to.

Replay

Capturing data (like a hash) and re-sending it later to pretend to be the authorized user. Prevented by timestamps/salts.
