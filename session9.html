01. SECURE DESIGN

Foundational principles for a hardened infrastructure.

Defense in Depth

Layered Security

Using multiple security controls to protect the same asset. If one layer fails, the others remain as obstacles.

  • Physical: Fences, locks, guards.
  • Network: Firewalls, IDS/IPS, VLANS.
  • Endpoint: EDR, AV, Hashing.
  • Data: Encryption, DLP.

"Think like an onion, not a walnut."

Secure by Default & Design

Secure by Default

Out-of-the-box settings are highly restrictive. (e.g., Guest account disabled, ports closed).

Secure by Design

Security is integrated into the architecture from day one, not bolted on at the end.

Fail-Safe

If a system fails, it defaults to a secure state (e.g., Electric door locks unlock for safety, but data access blocks).

Resilience & Availability

Redundancy

Active-Active or Active-Passive clusters to prevent a single point of failure (SPOF).

Load Balancing

Distributing requests across multiple servers to improve performance and availability.

02. PHYSICAL SECURITY

Protecting the "Tangible" assets.

Physical Controls: Access

Mantrap (Access Portal)

A small room with two doors. One must close and lock before the other opens. Prevents tailgating.

  • Authenticates users individually.
  • Prevents mass unauthorized entry.

Fencing & Lighting

Perimeter controls used as deterrents and delays.

  • Lighting: Detective and deterrent.
  • Fencing: Physical barrier.

Faraday Cages & TEMPEST

Faraday Cage

An enclosure made of conducting material used to block electromagnetic fields (RF interference).

Shielding

Protecting data cables from EMI (Electromagnetic Interference). Copper cables like UTP are vulnerable; fiber is immune.

  • Prevents data emanation.
  • TEMPEST: Standards for preventing data leakage through radiation.

Infrastructure Hardening

  • Immutable Systems: Systems that are replaced rather than updated to ensure consistency.
  • TPM (Trusted Platform Module): A chip providing hardware-level encryption keys.
  • Hardware Root of Trust: Ensuring the boot process starts with trusted code.
  • Infrastructure as Code (IaC): Using scripts to build environments identically every time.
