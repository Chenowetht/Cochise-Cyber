Event vs. Incident

The Event

Any observable occurrence in a system or network.
Example: A user logging in, a file being saved, or a ping request.

The Incident

An event that has a negative effect on the security of the system (CIA).
Example: A successful login from an unknown location, or a malware execution.

02. THE IR LIFECYCLE

The formal process for responding to a breach.

1. Preparation & 2. Identification

Preparation (Before)

Training, tools, policies, and establishing the CSIRT (Incident Response Team).

Identification (Now)

Determining if an incident has occurred. Analyzing logs, alerts, and user reports to find the "patient zero."

"Preparation is the most important—and most ignored—phase."

3. Containment

Isolation

Stopping the bleeding. Disconnecting a machine from the network or disabling a compromised account.

Strategy

Limiting the Blast Radius. We want to prevent lateral movement while preserving evidence for forensics.

4. Eradication & 5. Recovery

Eradication

Removing the root cause. Deleting malware, patching vulnerabilities, and resetting passwords.

Recovery

Restoring systems to normal operation. Re-imaging from clean backups and verifying security.

6. Lessons Learned

Post-Mortem Analysis

Meeting to discuss what went well and what didn't. This phase feeds directly back into Preparation.

  • Final Incident Report.
  • Policy and tool updates based on findings.

The Response Team (CSIRT)

  • Security Experts: To handle the technical analysis.
  • Legal/HR: To handle liability and personnel issues.
  • Public Relations (PR): To handle external communication.
  • Management: To provide authority and funding.
