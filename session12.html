01. AUTHENTICATION FACTORS

Moving beyond simple passwords to Multi-Factor Security.

The Five Factors of MFA

Knowledge: Something you know. (Password, PIN, Mother's maiden name).

Possession: Something you have. (Smart card, USB token, OTP app).

Inherence: Something you are. (Fingerprint, Retina scan, FaceID).

Location: Somewhere you are. (Geofencing, IP-based login restrictions).

Action: Something you do. (Typing cadence, gait analysis, signature).

Biometric Metrics

FAR & FRR

False Acceptance Rate (FAR): Unauthorized users get in. (Type II Error).

False Rejection Rate (FRR): Authorized users blocked. (Type I Error).

CER (Crossover Error Rate)

The point where FAR and FRR are equal. This is the primary metric for measuring the overall effectiveness of a biometric system.

"Lower CER = More Accurate System."

SSO & Federation

  • SSO (Single Sign-On): One set of credentials for multiple local systems.
  • Federation: Authenticating across different organizations (e.g., using Google to log into Zoom).
  • SAML: XML-based. Uses "Assertions." Great for web-based SSO.
  • OAuth: Authorization framework (using tokens).
  • OpenID Connect: Identity layer built on top of OAuth.

02. AUTHORIZATION MODELS

Defining how access decisions are made.

RBAC vs. ABAC

Role-Based (RBAC)

Access is based on job function or title. (e.g., "Managers can see payroll, Clerks cannot"). Static and easy to manage.

Attribute-Based (ABAC)

Access based on complex attributes: User (Clearance), Resource (Sensitivity), and Environment (Time/Location).

"ABAC is the backbone of dynamic Zero Trust policies."

DAC vs. MAC

Discretionary (DAC)

The data "Owner" decides who gets access. Highly flexible but risky. Used in standard Windows/Linux file systems.

Mandatory (MAC)

Access is decided by the system based on labels (Secret, Top Secret). Users cannot change permissions. Used in high-security Gov/Mil environments.

Governance & Lifecycle

  • Least Privilege: Only the permissions needed for the job.
  • Separation of Duties: Splitting a single high-risk task between multiple people.
  • Recertification: Periodic review of user access to ensure it is still appropriate.
  • Offboarding: Immediate account disablement upon employee departure.
