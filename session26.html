SY0-701 MASTERY

01. THE PBQ FORMAT

Going beyond multiple choice to prove practical skill.

Exam Strategy

Read, Skip, Return

PBQs appear at the very beginning of the exam. They are designed to drain your time and cause panic.

  • Skip First: Click through the PBQs and go straight to the multiple-choice.
  • Save Time: Complete all 70+ multiple-choice questions first to secure those points.
  • Return: Spend your remaining 30-45 minutes focusing on the simulations.

"Partial credit is often awarded. Never leave a PBQ completely blank."

PBQ Type: Firewall ACLs

1. PERMIT 10.0.1.5 80 ANY
2. DENY 10.0.1.0/24 ANY ANY
3. PERMIT 10.0.1.10 443 ANY
---
Result: Line 3 is never hit!

Rule Order Matters

You will likely be asked to configure an ACL to allow specific traffic while blocking others.

  • Top-Down: Rules are processed in order.
  • Implicit Deny: Assume all traffic is blocked unless a permit rule exists.
  • Specific to General: Place narrow rules (single IPs) above broad rules (subnets).

PBQ Type: Forensic Ordering

Order of Volatility

A classic "Drag and Drop" question. Order from Most Volatile to Least Volatile:

  1. CPU Cache & Registers
  2. Routing Tables, ARP Cache, Kernel Stats
  3. Memory (RAM)
  4. Temporary Files / Swap Space
  5. Hard Drive / SSD Data
  6. Remote Logs / Archive Media

02. LOG ANALYSIS

Identifying the attack from raw text.

Reading Web Logs

"GET /search.php?id=1 UNION SELECT user, password FROM users..."

SQL Injection (SQLi)

Look for keywords like UNION, SELECT, DROP, or ' OR 1=1--.

Cross-Site Scripting (XSS)

Look for <script> tags or alert() functions in the URL or form input.

HTTP Status Codes

200 OK

The attack or request was successful.

403 Forbidden

The request was blocked by a control (WAF/ACL).

404 Not Found

Often seen in directory traversal or scanning attempts.

PBQ Type: Matching RAID

Performance

RAID 0 (Striping only). Zero redundancy.

High Redundancy

RAID 1 (Mirroring). Requires 2 disks, identical copy.

The "Sweet Spot": RAID 5

Striping with Parity. Requires 3 disks. Can survive 1 disk failure. Provides both speed and safety.
