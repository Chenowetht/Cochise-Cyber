01. SENSITIVE DATA TYPES

Identifying what needs the most protection.

PII vs. PHI

PII

Personally Identifiable Information. Any data that can be used to distinguish or trace an individual's identity.
Examples: SSN, Full Name, Fingerprint, Passport #.

PHI

Protected Health Information. Medical records, health history, and insurance information. Covered under HIPAA.
Examples: Lab results, prescriptions, patient IDs.

"Privacy is about the right to control how this data is shared."

Classification Tiers

Public

No impact if leaked. Marketing materials, publicly listed contacts.

Sensitive

Moderate impact. Internal emails, project timelines.

Confidential

High impact. Trade secrets, PII/PHI, Financial records.

The Three States of Data

  • Data at Rest: Stored on a disk or cloud drive. (Protect with: Disk Encryption/AES).
  • Data in Transit: Moving over a network. (Protect with: TLS/SSL/IPsec).
  • Data in Use: Loaded in RAM or CPU for processing. (Protect with: Enclave memory/RAM encryption).

02. DATA PROTECTION TOOLS

Technical measures to prevent leakage.

DLP Implementation

The Sieve

Detects and stops sensitive data from leaving the network based on patterns (e.g., SSN format).

Types of DLP

  • Endpoint DLP: Prevents USB copying or printing sensitive docs.
  • Network DLP: Scans outgoing email and web traffic.
  • Cloud DLP: Monitors API calls and file sharing in SaaS apps.

Obfuscation & Privacy

Masking

Hiding parts of data.
Example: XXX-XX-1234

Tokenization

Replacing data with a non-sensitive 'token'. Used in credit card processing.

Anonymization

Removing all identifiers so the individual cannot be re-identified.

Data Sanitization

  • Wiping/Clearing: Overwriting data with 1s and 0s using software.
  • Degaussing: Using a magnetic field to destroy the data on HDDs (Magnetic media).
  • Destruction: Physically shredding, pulping, or incinerating the drive.

"Formatting is NOT sanitization; it only deletes the index."
