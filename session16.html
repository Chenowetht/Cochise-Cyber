01. THE SIEM ENGINE

Security Information and Event Management.

What is a SIEM?

Aggregation & Analysis

A centralized platform that collects data from many different sources across the infrastructure.

  • Aggregation: Pulling logs from firewalls, servers, endpoints.
  • Correlation: Linking related events to find a pattern.
  • Alerting: Notifying analysts when thresholds are met.

"The single source of truth for the security team."

SIEM Log Sources

Network

Firewall logs, IDS/IPS alerts, DNS queries.

Endpoint

OS event logs, Syslog, EDR telemetry.

Cloud

API logs (AWS CloudTrail), Identity logs (Azure AD).

02. SOAR & AUTOMATION

Security Orchestration, Automation, and Response.

SIEM vs. SOAR

SIEM (Detection)

"Something happened. Here is all the data about it."

SOAR (Action)

"I see that happened. I am going to automatically block that IP."

SOAR integrates with other security tools to execute actions automatically.

Playbooks & Runbooks

Playbooks (Strategic)

High-level process for a specific type of incident. (e.g., "Phishing Playbook").

Runbooks (Technical)

Step-by-step technical instructions to complete a specific task within a playbook.

"Automation uses runbooks to achieve consistent results at scale."

The Goal: Efficiency

  • MTTR (Mean Time to Respond): Reducing the time from detection to mitigation.
  • Alert Fatigue: Automating "noise" so analysts can focus on critical threats.
  • Consistency: Ensuring the same steps are taken every time an incident occurs.

Visualizing the SOC

Tracking trends over time.

KPIs (Key Performance Indicators)

  • Alert volume per day.
  • Percentage of alerts automated via SOAR.
  • False positive rate.
