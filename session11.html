01. SERVICE MODELS

Classifying cloud offerings by the "Level of Control."

SaaS vs. PaaS vs. IaaS

SaaS

Software as a Service. Fully managed apps. (Office 365, Salesforce). No infra management.

PaaS

Platform as a Service. For developers. (Azure App Service). OS is managed by provider.

IaaS

Infrastructure as a Service. Virtual hardware. (AWS EC2). You manage OS and apps.

"The higher you go, the less control (and responsibility) you have."

Shared Responsibility Model

Security OF the Cloud

Provider Responsibility: Physical data centers, host hardware, global infrastructure.

Security IN the Cloud

Customer Responsibility: Data, Identity Management, Network Config, Guest OS patching.

"Cloud providers are not magic; you are still responsible for your data."

Deployment Models

  • Public: Multi-tenant environment shared with anyone. (AWS, Azure).
  • Private: Single-tenant environment for one organization.
  • Hybrid: Combination of public and private (or on-prem).
  • Community: Shared between organizations with common concerns (e.g., healthcare).

02. VIRTUALIZATION

Abstracting hardware for efficiency and isolation.

Hypervisors (VMM)

Type 1 (Bare Metal)

Runs directly on the hardware. Fast and secure.
Examples: VMware ESXi, Hyper-V.

Type 2 (Hosted)

Runs on top of a host OS. Slower, used for testing.
Examples: VirtualBox, VMware Workstation.

Containerization

Lightweight & Portable

Unlike VMs, containers share the host's OS kernel. They package only the app and its dependencies.

  • Faster startup than VMs.
  • Uses fewer resources.
  • Security Risk: Escaping the container to the host kernel.

"The standard for modern microservices."

Advanced Cloud Controls

CASB

Cloud Access Security Broker. Enforces security policies between users and cloud apps.

Serverless

Function as a Service (FaaS). Provider manages servers; you just run code snippets.

MSSP

Managed Security Service Provider. Outsourcing security monitoring and management.
